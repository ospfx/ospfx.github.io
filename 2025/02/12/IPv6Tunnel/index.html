和朋友内网共享 IPv6 Tunnel

发表于 2025-02-12 阅读次数： Valine：

写在前面

最近朋友给我安利了一个很便宜的灵车 NAT VPS ，因为看着很便宜所以我也入手了一台。由于 HE.NET 免费提供的 IPv6 Tunnel 同一个公网IP只能创建一个隧道，我俩都想使用 HE 的 IPv6 Tunnel，下面提供了我们现用的解决方法。

隧道服务器配置

首先在 HE.NET 创建免费的 IPv6 隧道 https://tunnelbroker.net/new_tunnel.php ，填好服务器公网 IP 和隧道端点后点击 Example Configurations 选择示例的操作系统，参考HE下面的提示：当你的设备在防火墙后时, 使用DHCP分配给你的内网地址而不是你提供的HE的公网地址。例如我这里使用的是Debian系统，我的配置如下(编辑/etc/network/interfaces)：

配置好之后使用ifup he-ipv6命令来启用它，如果没有的ifupdown命令的话使用其他命令也是可以的，例如networkctl reload。
ping6 google.com 测试 ok 。

网关配置

接下来给隧道服务器配置 HE 分配的地址作为内网的网关：

重启网卡或者重启eth1 ifdown eth1 && ifup eth1。这里直接配置地址就可以了，默认 ipv6 路由会从 he-ipv6 接口出去。

打开IPv6转发

请确保你的 cat /etc/sysctl.conf 中有以下内容net.ipv6.conf.all.forwarding=1 ，如果没有则写入该配置echo "net.ipv6.conf.all.forwarding=1" >> /etc/sysctl.conf，然后执行sysctl -p

内网服务器配置

因为机器配置很低，而且只有朋友的机器需要使用这个v6，故此直接在朋友的服务器上手写ipv6地址配置/etc/network/interfaces。
